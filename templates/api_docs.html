Dokumentacja API

Wszystkie endpointy wymagają nagłówka X-API-Key. Klucz wygenerujesz w Ustawieniach.

Base URL: http://<adres>:42569/api/v1
Autoryzacja: X-API-Key: hf_twoj_klucz
Content-Type: application/json (dla POST/PUT)

System

GET /api/v1/status

Ogólny status systemu.

Odpowiedź:

{
  "success": true,
  "version": "1.0.0",
  "uptime_seconds": 3600,
  "upnp_online": true,
  "port_count": 5,
  "router_ip": "192.168.55.1"
}

Porty

GET /api/v1/ports

Lista wszystkich portów.

Odpowiedź:

{
  "success": true,
  "ports": [
    {
      "id": 1,
      "protocol": "TCP",
      "external_port": 25565,
      "internal_address": "192.168.55.150",
      "internal_port": 25565,
      "description": "Minecraft"
    }
  ]
}
POST /api/v1/ports

Dodaj nowy port. Automatycznie dodaje do UPnP.

Body (JSON):

{
  "protocol": "TCP",        // TCP, UDP lub TCP/UDP
  "external_port": 25565,
  "internal_address": "192.168.55.150",
  "internal_port": 25565,
  "description": "Minecraft"
}

Odpowiedź (201):

{
  "success": true,
  "id": 1,
  "message": "Port dodany"
}
DELETE /api/v1/ports/:id

Usuń port. Automatycznie usuwa z UPnP.

Odpowiedź:

{
  "success": true,
  "message": "Port usunięty"
}

UPnP

GET /api/v1/upnp/status

Sprawdź czy UPnP jest online.

Odpowiedź:

{
  "success": true,
  "online": true
}
GET /api/v1/upnp/mappings

Pobierz aktualną listę port mappings bezpośrednio z routera (live).

Odpowiedź:

{
  "success": true,
  "mappings": [
    {
      "protocol": "UDP",
      "external_port": 34632,
      "internal_port": 41641,
      "internal_address": "192.168.55.173",
      "description": "tailscale-portmap",
      "enabled": true
    }
  ]
}
POST /api/v1/upnp/restart

Wymuś restart UPnP na routerze (wyłącz → włącz).

Odpowiedź:

{
  "success": true,
  "message": "UPnP zrestartowany"
}

Ustawienia

GET /api/v1/settings

Pobierz aktualne ustawienia.

Odpowiedź:

{
  "success": true,
  "settings": {
    "auto_add_ports": false,
    "auto_restart_upnp": true,
    "logodev_api_key": null
  }
}
PUT /api/v1/settings

Zmień ustawienie.

Body (JSON):

{
  "key": "auto_restart_upnp",   // auto_add_ports, auto_restart_upnp, logodev_api_key
  "value": true                  // true/false dla bool, string dla reszty
}

Odpowiedź:

{
  "success": true,
  "message": "Zapisano"
}

Kody błędów

KodZnaczenie
200OK
201Utworzono (nowy port)
400Błędne dane wejściowe
401Brak klucza API
403Nieprawidłowy klucz API
404Nie znaleziono (np. port o danym ID)
500Błąd serwera

Przykład (curl)

# Status
curl http://localhost:42569/api/v1/status -H "X-API-Key: hf_twoj_klucz"

# Dodaj port
curl http://localhost:42569/api/v1/ports \
  -H "X-API-Key: hf_twoj_klucz" \
  -H "Content-Type: application/json" \
  -X POST \
  -d '{"protocol":"TCP","external_port":25565,"internal_address":"192.168.55.150","internal_port":25565,"description":"Minecraft"}'

# Usuń port
curl http://localhost:42569/api/v1/ports/1 -H "X-API-Key: hf_twoj_klucz" -X DELETE

# Restart UPnP
curl http://localhost:42569/api/v1/upnp/restart -H "X-API-Key: hf_twoj_klucz" -X POST
